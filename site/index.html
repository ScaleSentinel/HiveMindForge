Comparação entre TimescaleDB e Druid

Critério Druid TimescaleDB
Tipo de Banco Colunar, OLAP Extensão PostgreSQL
Escalabilidade Distribuído Escala vertical/horizontal
Latência de Leitura Baixa para OLAP Boa para TimeSeries
Latência de Escrita Batch-oriented Suporta ingestão contínua
Suporte a JSON ✅ Sim ✅ Via JSONB

Configurando TimescaleDB no Docker

docker run -d --name timescaledb \
  -p 5432:5432 \
  -e POSTGRES_USER=admin \
  -e POSTGRES_PASSWORD=admin \
  -e POSTGRES_DB=metrics \
  timescale/timescaledb:latest-pg15

Criação da Tabela

CREATE TABLE metrics (
    time TIMESTAMPTZ NOT NULL,
    device_id TEXT NOT NULL,
    value DOUBLE PRECISION
);
SELECT create_hypertable('metrics', 'time');

Configurando Druid no Docker

docker run -d --name druid \
  -p 8888:8888 \
  -p 8082:8082 \
  -p 8090:8090 \
  apache/druid:latest

Inserindo Dados via API

curl -X POST 'http://localhost:8082/druid/v2/sql' \
  -H 'Content-Type: application/json' \
  -d '{"query": "INSERT INTO metrics VALUES (NOW(), '\''sensor-01'\'', 23.5)"}'
